Conceptos claves
Los siguientes son términos y conceptos centrales para interactuar con el sistema Loro.
Proyecto Es el elemento integral de organización de tu trabajo. Un proyecto es el conjunto de componentes de desarrollo asociados a una tarea o misión concreta. Algunos proyectos de demostración vienen incluidos con el sistema y otros se pueden incorporar mediante algunos mecanismos para ello. Loro destina una ventana de trabajo para cada proyecto.
Espacio de trabajo Es el conjunto de proyectos que pueden accederse en un momento dado. "Acceder" un proyecto significa abrirlo para trabajar en él, o simplemente referir elementos contenidos en él (por ejemplo, invocar un cierto algoritmo). Para abrir un proyecto del espacio de trabajo, utiliza el comando Ctrl-O. Para instalar un proyecto en el espacio de trabajo, utiliza el comando Ctrl-I.
Edición Se refiere a la escritura de código fuente en el lenguaje.  Loro destina un "editor" (ventana) para cada componente del proyecto. Toda edición o despliegue de código fuente siempre se hace con lo que se suele llamar "coloreamiento sintáctico", es decir, con utilización de colores y resaltados para distinguir los diferentes elementos en el programa.
Compilación Significa hacer que el sistema revise lo que está escrito para verificar que se respeten las reglas del lenguaje en los aspectos de forma y contenido. La forma se refiere a la estructura que un programa debe tener (por ejemplo, que después de una palabra clave si venga una expresión y luego una palabra clave entonces); y el contenido se refiere a que el programa respete ciertas reglas de consistencia a un mayor nivel (por ejemplo, que a una variable de tipo cadena no se le asigne una expresion de tipo entero, o que un arreglo no se subindice con un valor real como 3.14). Ante cualquier problema con el código, el sistema despliega un mensaje de aviso y selecciona el área de texto comprometida con el error en la ventana de edición correspondiente.
Ejecución Se refiere a llevar a cabo las acciones o instrucciones de un algoritmo.  El entorno ofrece facilidades para ejecutar cualquier algoritmo, hacer la captura interactiva de sus posibles entradas, también la ejecución de fragmentos de código a través del intérprete interactivo, y la modalidad de paso-a-paso como ayuda para la depuración del código.
Prueba de unidades Loro incluye facilidades para poner en práctica la aplicación de pruebas a nivel de algoritmos individuales como a nivel de proyecto completo. 
Documentación ¿Qué se tiene?, ¿para qué sirve?, ¿cómo se usa?  y ¿bajo qué condiciones?, son algunas de las consideraciones que se presentan permanentemente en la práctica de la programación. Puesto que la mayor parte del software se contruye sobre software ya existente, se vuelve imprescindible contar con toda la documentación posible que resuelva estas preguntas. Loro genera automáticamente esta documentación en un formato HTML (es decir, en estilo página de Internet) con base en los elementos del lenguaje destinados para tal fin.
