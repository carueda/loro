Key concepts
The following are some central terms and concepts in the Loro Programming System.
Project All programming in the environment is organized around projects. A project is the set of development components associated to a specific task.
Workspace The set of projects that are currently available in the system. Some demo projects are included and others can be easily installed.
Edition An editor window is dispatched for each component in a project. Syntax highlighting is used so lexical and grammar elements are easily recognized.
Compilation Your code must be checked to verify it follows the rules of the language.
Execution Any algorithm or code fragment can be run at any time. Expressions can also be evaluated through the interactive interpreter. Any execution or evaluation can be done in a step-by-step fashion.
Unit testing The environment also provides the mechanisms to allow you follow a test-driven development discipline. Individual algorithms and whole projects can be easily tested.
Documentation What is available?, what is used for?, how is used?, and under what conditions?, are some of the considerations that are part of the practice of programming. Loro generates HTML documentation based on the elements in the language for that purpose. A simple HTML browser is included to easily navigate the generated documentation.
